Como Funciona o AgroCalc

Arquitetura: CPU, Memória e Armazenamento

Qualquer computador — incluindo o servidor que roda o AgroCalc — funciona com três componentes fundamentais que trabalham juntos:

CPU (Processador)
Executa os cálculos matemáticos do sistema:
juros compostos, determinantes, conversões de base numérica. Cada fórmula Python se torna instruções que a CPU executa em microssegundos.
Memória RAM
Armazena temporariamente os dados enquanto você usa o sistema: os valores digitados nos formulários, as matrizes de talhões, os resultados calculados — tudo em memória volátil (some ao fechar).
Armazenamento (HD/SSD)
Guarda permanentemente os arquivos do projeto: os arquivos .html, .css, .js e .py que compõem o AgroCalc e os gráficos exportados pelo Matplotlib.

Linha do Tempo do Projeto

Etapa 1
Planejamento — Wireframe e Estrutura HTML
Definição dos módulos, esboço das telas no papel (wireframe), criação da estrutura de pastas e dos arquivos base do projeto.
Etapa 2
CSS Responsivo e Modo Escuro
Aplicação do CSS com Grid e Flexbox para responsividade mobile, implementação do toggle de dark mode com CSS e JavaScript.
Etapa 3
Scripts Python e Integração com Forms
Desenvolvimento dos módulos Python (custo, financiamento, conversor, matriz NumPy, blend NPK, gráficos Matplotlib) e integração com os formulários HTML via Flask.
Etapa 4
Testes, Relatório e Apresentação
Testes de responsividade em diferentes dispositivos, produção do relatório do projeto e slides de apresentação. Deploy no GitHub Pages ou Netlify.

Glossário de Termos Técnicos

  • HTML (HyperText Markup Language) — Linguagem de marcação usada para estruturar o conteúdo das páginas web. Define elementos como títulos, parágrafos, formulários e tabelas.
  • CSS (Cascading Style Sheets) — Linguagem de estilos que controla a aparência visual do HTML: cores, fontes, espaçamentos, layout e responsividade.
  • JavaScript — Linguagem de programação executada no navegador que adiciona interatividade às páginas sem precisar recarregá-las.
  • Python — Linguagem de programação de alto nível usada para os cálculos matemáticos do AgroCalc: funções afins, juros compostos, álgebra linear e geração de gráficos.
  • NumPy — Biblioteca Python para computação numérica. Usada no AgroCalc para representar a matriz de produtividade dos talhões e resolver sistemas lineares com numpy.linalg.solve.
  • Matplotlib — Biblioteca Python para geração de gráficos e visualizações. Usada no AgroCalc para exportar gráficos de custo e financiamento como imagens.
  • Sistema binário (base 2) — Sistema de numeração com apenas dois dígitos (0 e 1). É a linguagem nativa dos computadores, pois representa os estados elétrico-lógicos dos transistores.
  • Hexadecimal (base 16) — Sistema de numeração com 16 símbolos (0–9 e A–F). Muito usado em programação para representar valores binários de forma mais compacta (ex.: cores CSS como #2e7d32).
  • Sistema linear — Conjunto de equações lineares com múltiplas incógnitas. No AgroCalc, um sistema 3×3 é usado para descobrir quanto de cada fertilizante usar para atingir o alvo NPK.
  • Determinante — Valor escalar calculado a partir de uma matriz quadrada. Se det ≠ 0, o sistema linear tem solução única. Se det = 0, o sistema não tem solução única (dependência linear).
  • NPK — Sigla para Nitrogênio (N), Fósforo (P) e Potássio (K), os três macronutrientes primários dos fertilizantes agrícolas.
  • Talhão — Subdivisão de uma propriedade rural usada para organizar o manejo agrícola. Cada talhão pode ter características de solo e produtividade diferentes.
  • Função afim — Função matemática da forma f(x) = ax + b. No AgroCalc, usada para calcular o custo de produção: C(x) = custo_fixo + custo_variável × x.
  • Juros compostos (Tabela Price) — Sistema de amortização de empréstimos onde os juros incidem sobre o saldo devedor. Gera parcelas fixas ao longo do tempo. Fórmula: PMT = PV × [i(1+i)ⁿ] / [(1+i)ⁿ − 1].
  • Responsividade — Capacidade de uma página web de se adaptar a diferentes tamanhos de tela (desktop, tablet, celular). Implementada com CSS Grid e Flexbox.
